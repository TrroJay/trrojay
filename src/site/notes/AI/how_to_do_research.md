## 1.1 “上瘾”
- Troy
	- 这几天跑代码跑得好累，和朋友的聊天框空了好多天（感觉自己还是有一些回避，也在怄气），偷偷鸽了一个约定，也提不起兴趣和你来对话。
	- 与读论文相比，跑代码更容易让人上瘾，我很好奇是为什么，我先描述一下我跑代码时的状态。在修改代码的时候我的注意力是高度集中的，我不会想起过去、也不会想到未来，我所关注的只有眼前的代码以及运行代码的结果。在跑代码时我会感觉枯燥，如果反复出 bug 也会觉得烦躁，等代码出结果的时候会感到无聊，想要填充这段时间。但我的疲惫感不容易袭来，只有当我缓过神，尝试把注意力集中到其他事情上的时候，我才发现：原来我已经很累了。
	- 回过神想一想，我在跑代码的时候更多时间在做尝试，而不是做思考。做尝试指的是手头不断切换着代码文件、不断问 AI、不断调整参数打断点，最后运行代码开始摸奖。如果有一个地方出了 bug，那就从头来过、、、做思考更多地是在搭框架，在理思路。也许在没有 AI Agent 的时代，具体实现的过程也需要自己思考，但是在 AI Agent 的帮助下，我只需要提要求，看代码，debug 就可以了。
	- 让自己“忙起来”是不是就是“上瘾”的原因呢？当我有无数个细细碎碎的目标需要优化，我也有手段来优化，我也能立刻看到结果，这就是一个密集奖励的优化任务呀。（似乎有一点像玩游戏了）

- Rion
	- 是不是跑代码还挺 Se 的？所做即所得，一切均可视。
	- 我来帮你总结几个词
		- 目标拆解（模块化任务）
		- 即时反馈
		- 流程化、套路化
		- 主客体交互
		- 伪难度，难度匹配

| **特性**   | **读论文 (Deep Thinking)** | **跑代码 (Trial & Error)**  |
| -------- | ----------------------- | ------------------------ |
| **反馈周期** | 极长（可能读几个小时才有一点启发）       | 极短（几秒钟到几分钟看一次 Log）       |
| **奖励机制** | 延迟奖励（建立知识体系）            | **随机变量奖励**（像老虎机，下次可能就通了） |
| **认知摩擦** | 高（需要调动抽象逻辑，极度耗能）        | 低（尤其是有 AI 辅助时，更多是执行和微调）  |
| **互动感**  | 较低（作者的思路不一定能立刻理解）       | 极高（代码改了立刻生效，哪怕报错也是一种互动）  |
