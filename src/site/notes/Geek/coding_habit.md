# 1 模型训练速度（小模型）
## 1.1 性能瓶颈
### 1.1.1 GPU 与 CPU 的交互
- 在模型训练过程中，GPU 能够实现高速的计算，但由于代码实现的原因，其需要等待 CPU 指令、等待 CPU 传输数据等，这些步骤严重拖慢了模型的训练速度。
### 1.1.2 代码逻辑低效
- 调用过多的循环，使用复杂的判断语句……
## 1.2 提升训练速度的方法
#### Compile 的使用
```
使用 torch.compile 方法将一个函数进行预编译，官方文档指出可以提升20%性能
```
#### 将张量全部转移到 GPU 上
- 由于训练的是显存需求小的强化学习任务，可以将所有 tensor 都转移到 GPU 上。
```
将to_td函数中的所有.cpu()向量全部移动到GPU上
```
#### 优化循环结构
- 损失函数计算中使用 for 循环速度慢
```
在模型损失函数计算与更新的代码中，将for循环逐个计算转化为向量相乘
```
#### 减少 if 判断的使用
```
使用torch中的where方法进行判断
```
